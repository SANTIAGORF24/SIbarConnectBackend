# Documentación del .gitignore

## Descripción

El archivo `.gitignore` especifica qué archivos y directorios deben ser ignorados por Git, evitando que se suban al repositorio archivos sensibles, temporales, o generados automáticamente.

## Estructura y Organización

El `.gitignore` está organizado en secciones temáticas para facilitar su mantenimiento:

### 🔐 **Archivos de Configuración Sensibles**
```gitignore
.env
.env.*
.env.local
config.ini
secrets.json
credentials.json
*.pem
*.key
*.crt
```

**Propósito**: Evitar que credenciales, tokens, contraseñas y certificados se suban al repositorio

**⚠️ CRÍTICO**: Estos archivos NUNCA deben estar en el repositorio por seguridad

### 🐍 **Entornos Virtuales Python**
```gitignore
venv/
env/
.venv/
.virtualenv/
.conda/
```

**Propósito**: Los entornos virtuales son específicos de cada desarrollador y máquina

**Razones para excluir**:
- Archivos binarios pesados (cientos de MB)
- Específicos del sistema operativo
- Se recrean con `pip install -r requirements.txt`

### 🔥 **Archivos Python Compilados**
```gitignore
__pycache__/
*.pyc
*.pyo
*.pyd
```

**Propósito**: Python genera estos archivos automáticamente para mejorar el rendimiento

**Características**:
- Se regeneran automáticamente
- Específicos de la versión de Python
- No aportan valor al código fuente

### 📦 **Empaquetado y Distribución**
```gitignore
build/
dist/
*.egg-info/
wheels/
```

**Propósito**: Archivos generados durante el proceso de empaquetado de Python

### 🧪 **Testing y Coverage**
```gitignore
.pytest_cache/
.coverage
htmlcov/
.tox/
```

**Propósito**: Archivos generados por herramientas de testing y coverage

### 🗄️ **Bases de Datos**
```gitignore
*.db
*.sqlite
*.sql
*.dump
```

**Propósito**: Archivos de base de datos locales y dumps

**Nota**: En desarrollo se suele usar SQLite local, en producción PostgreSQL/MySQL

### 📝 **Logs y Archivos Temporales**
```gitignore
*.log
tmp/
*.tmp
*.swp
```

**Propósito**: Archivos de log y temporales que se generan durante la ejecución

### 💻 **IDEs y Editores**
```gitignore
.vscode/
.idea/
*.sublime-workspace
```

**Propósito**: Configuraciones específicas de editores e IDEs

**Beneficio**: Cada desarrollador puede usar su IDE preferido

### 🖥️ **Sistemas Operativos**
```gitignore
.DS_Store     # macOS
Thumbs.db     # Windows
*~            # Linux
```

**Propósito**: Archivos generados automáticamente por diferentes sistemas operativos

## Validación del .gitignore

### ✅ **Verificar que funciona correctamente**

```bash
# Ver archivos que Git está trackeando
git ls-files

# Ver archivos ignorados
git status --ignored

# Verificar si un archivo específico está ignorado
git check-ignore archivo.env
```

### 🔍 **Comandos útiles**

```bash
# Ver todos los archivos ignorados en el directorio actual
git status --ignored --short

# Forzar agregar un archivo ignorado (¡cuidado!)
git add -f archivo_ignorado.py

# Dejar de trackear un archivo ya committeado
git rm --cached archivo_sensible.env
```

## Mejores Prácticas

### ✅ **Hacer**

1. **Añadir .gitignore antes del primer commit**
2. **Mantener organizadas las secciones**
3. **Comentar patrones complejos**
4. **Usar patrones específicos en lugar de genéricos**
5. **Revisar periódicamente y actualizar**

### ❌ **No hacer**

1. **Nunca commitear archivos con credenciales**
2. **No ignorar archivos importantes del proyecto**
3. **No usar patrones demasiado amplios**
4. **No ignorar el propio .gitignore**

## Patrones Comunes

### **Comodines**
```gitignore
*.log          # Todos los archivos .log
temp*          # Archivos que empiecen con "temp"
**/node_modules # node_modules en cualquier subdirectorio
```

### **Directorios**
```gitignore
logs/          # Directorio logs y todo su contenido
/build         # Solo directorio build en la raíz
```

### **Excepciones**
```gitignore
*.log          # Ignorar todos los .log
!important.log # EXCEPTO important.log
```

## Casos Específicos del Proyecto

### **FastAPI/Uvicorn**
```gitignore
uvicorn.log
static/        # Si se generan archivos estáticos
```

### **SQLAlchemy/Alembic**
```gitignore
# Descomenta si quieres ignorar migraciones auto-generadas
# alembic/versions/
```

### **Docker**
```gitignore
.dockerignore
docker-compose.override.yml
```

### **Environment Files**
```gitignore
.env                    # Desarrollo local
.env.production        # Producción
.env.staging          # Staging
.env.testing          # Testing
```

## Validación de Seguridad

### **Verificar que no hay archivos sensibles**

```bash
# Buscar posibles archivos de configuración
find . -name "*.env*" -o -name "config.*" -o -name "secret*"

# Verificar que están ignorados
git check-ignore .env config.ini secrets.json
```

### **Audit de archivos trackeados**

```bash
# Buscar archivos que podrían contener secretos
git ls-files | grep -E "\.(env|config|secret|key|pem)$"

# Revisar historial para secretos accidentalmente committeados
git log --all --full-history -- .env
```

## Mantenimiento

### **Actualización periódica**

1. **Revisar nuevos archivos generados** durante el desarrollo
2. **Añadir patrones para nuevas herramientas**
3. **Mantener organizadas las secciones**
4. **Documentar patrones complejos**

### **Limpieza de archivos ya trackeados**

Si accidentalmente se committearon archivos que deberían estar ignorados:

```bash
# Para un archivo específico
git rm --cached archivo_sensible.env

# Para un directorio
git rm -r --cached directorio/

# Commit los cambios
git commit -m "Remove sensitive files from tracking"
```

## Integración con CI/CD

### **Validación en pipeline**

```yaml
# En .github/workflows/
- name: Check for sensitive files
  run: |
    if git ls-files | grep -E "\.(env|key|pem)$"; then
      echo "Sensitive files found in repository!"
      exit 1
    fi
```

Este `.gitignore` mejorado proporciona una protección completa para el proyecto, evitando problemas comunes de seguridad y mantenimiento.
